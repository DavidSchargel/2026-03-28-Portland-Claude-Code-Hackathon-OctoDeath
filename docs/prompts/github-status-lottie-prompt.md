# GitHub Status Lottie App — Implementation Prompt

Build a TanStack Start website that displays a Lottie animation reflecting the live status of github.com. Create a server-side API route within TanStack Start that proxies a fetch to `github.com` to avoid CORS — the client polls this endpoint every 10 seconds. Based on the response, the client plays one of three Lottie animations located in `src/assets/`: `lottie-github-up.json` when GitHub is reachable, `lottie-github-down.json` when it is not, and `lottie-github-victory.json` when GitHub comes back online after having been down. The victory animation plays for exactly 5 seconds, then automatically transitions back to `lottie-github-up.json`. Choose an appropriate Lottie renderer package for the project. Style the UI with plain CSS only — no CSS frameworks.

At the top of the page, include a toggle button that simulates a GitHub outage independent of the real poll. The button initially reads "GitHub Down!" — pressing it immediately plays `lottie-github-down.json` and changes the button label to "GitHub Up!". Pressing "GitHub Up!" immediately plays `lottie-github-victory.json`, and after 1 second the button resets to "GitHub Down!" again. The simulated state and the real poll state should coexist cleanly: when the manual toggle is not active, the UI reflects the live polling result; the toggle overrides the display temporarily and then yields back to the real status once the button has fully cycled back to "GitHub Down!".
