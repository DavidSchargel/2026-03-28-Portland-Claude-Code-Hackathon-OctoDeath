GitHub Status
Lottie Monitor

A live GitHub availability monitor built on TanStack Start. Polls github.com every 10 s via a server-side CORS proxy and plays one of three Lottie animations: up, down, and a 5-second victory sequence when GitHub recovers. Manual override lets you simulate outages without waiting for real failures.

Feature Work
featureServer-side GitHub proxy via createServerFn

fetchGitHubStatus issues a HEAD request to https://github.com with a 5-second AbortSignal.timeout. Returns { up: boolean }. Co-located in GitHubStatus.tsx — no separate api/ route file.

featureThree-state animation machine: up / down / victory

displayStatus = manualOverride ?? realStatus. Victory fires on a real down→up flip (detected via prevRealStatus.current) for 5 seconds, or on manual “GitHub Up!” press for 1 second. Each trigger owns a separate timer ref.

featureManual outage toggle with clean state coexistence

“GitHub Down!” → “GitHub Up!” → auto-reset after 1 s. Real poll runs continuously in the background; manualOverride shadows it temporarily and clears when the cycle completes.

featureLazy-loaded Lottie with SSR Suspense boundary

lottie-react wrapped in React.lazy() + <Suspense> to avoid crashing TanStack Start’s SSR pass. .lottie-placeholder fallback keeps layout stable during hydration.

Decisions

Decision Log

Key choices made during this build — 4 commits by David Schargel today on trying-01. The reasoning that evaporates fastest when context is lost.

Use createServerFn, not a separate API route file

TanStack Start’s createServerFn lets server logic live in the same file as the component that calls it. fetchGitHubStatus stays in GitHubStatus.tsx — no routing configuration, no separate api/ directory. The alternative (a dedicated route under src/routes/api/) was rejected as unnecessary ceremony for a single function.

architectureTanStack Start
Lazy-load lottie-react via React.lazy() to survive SSR

lottie-react accesses the DOM at module evaluation time. A direct import throws on TanStack Start’s server render pass. The fix: React.lazy() defers the import until client hydration. A Suspense boundary handles the loading state. Called out explicitly in CLAUDE.md as a non-obvious requirement.

SSRgotcha
Two separate timer refs — victoryTimerRef and manualTimerRef

Victory plays in two scenarios: a real status flip (5 s, owned by victoryTimerRef) and a manual “GitHub Up!” press (1 s, owned by manualTimerRef). Consolidating into one timer would create ambiguity over which duration to apply when both paths are active simultaneously. Separate refs give each scenario exclusive ownership.

state design
Light-mode only — dark mode CSS deliberately removed

Rather than shipping a half-finished dual-mode experience, dark mode variables and media queries were stripped from styles.css. One polished mode beats two mediocre ones. ThemeToggle was left in the layout (scaffold default) but is effectively a no-op for dark/auto. Documented in CLAUDE.md.

styling
shellComponent vs component in __root.tsx

The root route uses shellComponent: RootDocument (SSR shell, owns <html>/<head>/<body>), not component. This is the TanStack Start SSR pattern. Using component here would break the document structure. This distinction doesn’t exist in plain TanStack Router (SPA-only) and is easy to get wrong when coming from that background.

SSRrouting
Tailwind utilities + plain CSS classes — mixed approach

The seed prompt asked for plain CSS only, but the scaffold included Tailwind 4. The resolution: Tailwind utility classes handle the boilerplate layout/spacing inherited from the scaffold; all new feature styles (.github-status, .toggle-btn, .lottie-container, .status-label) are plain CSS appended to styles.css.

Mental Model

What To Hold In Your Head

The seven things you need to remember to work effectively on this codebase.

01

manualOverride always wins

displayStatus = manualOverride ?? realStatus. The real poll runs continuously but manualOverride shadows it completely until cleared to null. When the toggle fully cycles, it calls setManualOverride(null) and real status resumes.

02

Victory has two triggers, two durations

Real down→up flip fires setManualOverride('victory') for 5 seconds via victoryTimerRef. Manual “GitHub Up!” fires the same state for 1 second via manualTimerRef. Different refs, different durations — don’t consolidate them.

03

lottie-react must be lazy-loaded

lottie-react accesses the DOM at import time. TanStack Start runs SSR. Direct import = server crash. Pattern: React.lazy() + <Suspense>. Moving the import or adding another animation library will silently break SSR.

04

createServerFn is the server primitive

Server-only logic lives in component files via createServerFn, not in separate api/ routes. Any future server-side work (auth, secrets, DB) should follow this pattern. The compiled output stays co-located.

05

routeTree.gen.ts is auto-generated

Never edit src/routeTree.gen.ts manually. The TanStack Router Vite plugin regenerates it on every dev server start. Drop a file in src/routes/ and the route registers automatically.

06

shellComponent is not component

The root route uses shellComponent (owns <html>/<head>/<body>), not component. Wrong choice here breaks the server-rendered document structure. This distinction doesn’t exist in plain TanStack Router.

07

Dark mode is off; Lottie copies are orphans

Dark CSS vars were stripped. ThemeToggle dark/auto states do nothing visually. Only octocat-death/src/assets/ Lottie files are used at runtime — the copies in root assets/, root src/assets/, and octocat-death/assets/ are unused orphans.

Cognitive Debt

Hotspots

Areas where understanding is weakest or rationale is undocumented. Each has a concrete, actionable suggestion.

ThemeToggle cycles states that have no CSS behind them
High

ThemeToggle.tsx is fully wired: it cycles light/dark/auto, persists to localStorage, and applies class="dark" to <html>. But dark mode CSS variables were stripped from styles.css. Pressing “Dark” applies the class — nothing in the stylesheet responds. This is a silent broken feature invisible to anyone unfamiliar with the CLAUDE.md note.

Fix: Either (a) remove ThemeToggle from __root.tsx and the Header to be honest about scope, or (b) re-add dark mode @media (prefers-color-scheme: dark) CSS variables to styles.css. Document the outcome clearly in CLAUDE.md. Source: ThemeToggle.tsx + CLAUDE.md line about light-mode only.

Two-timer-ref interaction is undocumented
Medium

The interaction between victoryTimerRef and manualTimerRef in GitHubStatus.tsx (L29–31, L78–88) is subtle. If “GitHub Up!” is pressed while a real-status victory is already playing, handleToggle sets a new manualOverride('victory') without clearing victoryTimerRef — two timers are now racing to call setManualOverride(null). The behavior is likely acceptable but the intent is unwritten.

Fix: Add a block comment above the timer refs explaining: (1) what each owns, (2) expected behavior when both are active, (3) why they’re not consolidated. Source: GitHubStatus.tsx L29–31, L78–88.

Confetti animation added but not yet wired into GitHubStatus.tsx
Low

Commit f8248b4 added confetti.json to octocat-death/assets/ for “celebratory effects.” The core component GitHubStatus.tsx has local uncommitted modifications, likely the in-progress integration. But the confetti asset is in the octocat-death/assets/ directory (which is separate from the runtime src/assets/) — the import path will need updating when wiring it in.

Fix: Move confetti.json to octocat-death/src/assets/ before importing it. Then commit the GitHubStatus.tsx changes that integrate it. Clarify whether confetti plays as an overlay on victory or replaces the victory Lottie. Source: f8248b4, git status (GitHubStatus.tsx modified).

Zero tests despite Vitest being configured
Low

Vitest 3 is installed and npm run test runs, but no test files exist. The most valuable tests would cover the state machine logic in GitHubStatus.tsx — specifically the manualOverride / realStatus interaction, which is the most complex behavior and easiest to accidentally break during refactoring.

Fix: Create src/components/GitHubStatus.test.tsx. Priority cases: (1) real down→up flip triggers 5s victory, (2) manual toggle overrides real status, (3) manual cycle resets after 1s, (4) unmount cleanup cancels both timers. Source: package.json devDependencies, GitHubStatus.tsx.

Lottie JSON files duplicated across three directories
Low

The same three Lottie JSON files appear in assets/ (root), src/assets/ (root), and octocat-death/assets/. Only octocat-death/src/assets/ is imported at runtime (via GitHubStatus.tsx L8–10). The other locations are orphans from earlier experimentation and add noise to the file tree.

Fix: Delete root assets/, root src/assets/, and octocat-death/assets/. Verify no other import paths reference them first. Source: file tree + GitHubStatus.tsx L8–10.

Header and Footer are unmodified TanStack boilerplate
Low

Header.tsx links to TanStack’s X and GitHub accounts. Footer.tsx says “Your name here. All rights reserved.” and links to TanStack socials. about.tsx says “A small starter with room to grow.” None of this reflects the octocat-death project and will read as unfinished to any outside viewer.

Fix: Customize or remove Header, Footer, and About page. If the project is single-purpose, consider dropping the multi-page navigation entirely and keeping only a minimal footer attribution. Source: Header.tsx, Footer.tsx L8, about.tsx L15.
